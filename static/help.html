Quickstart Deployment Guide

Get your artwork display engine running in minutes using Docker. This is the recommended method for most users as it bundles all dependencies (Python, Database, and Web Server) into a single, isolated container.

Prerequisites

  • Docker: Installed and running on your host machine.
  • Docker Compose: (Included with Docker Desktop on Windows/Mac, may need separate install on Linux).

Installation Commands

Open your terminal and run the following sequence to clone the repository and start the engine:

# 1. Clone the repository
git clone https://github.com/your-repo/art-engine.git

# 2. Enter the directory
cd art-engine

# 3. Build and start the container in detached mode
docker compose up -d --build
Data Persistence: Your database and artwork library are stored in persistent Docker volumes (mapped to the ./Artwork and ./artwork.db files on your host). This means you can safely update the application via git pull and a rebuild without losing your settings or uploaded images.

First Steps

Once the container is healthy, you can access the different parts of the system via your browser:

  • Admin Dashboard: http://localhost:8000/admin — Start here to create your first playlist and upload high-resolution images.
  • Main Display: http://localhost:8000/ — Point your TV or monitor browser here to begin the artwork loop.
  • Mobile Remote: http://localhost:8000/remote — Access this on your phone to control the playback from your couch.

AI Curator Agent & Review Workflow

The engine features an autonomous AI pipeline that acts as a digital curator, analyzing your artwork and generating high-quality metadata for the "Museum Placard" display.

The .env File & API Keys

Before running the system via Docker, you must create a .env file in the project root to store your credentials. This file is ignored by version control to keep your keys secure.

# Required for the AI Curator Agent
GEMINI_API_KEY=your_free_key_here

Get a Key: You can obtain a free API key from Google AI Studio. The free tier is more than sufficient for standard personal use.

The AI Curator Pipeline

When you upload an image via the Admin Dashboard, the following automated workflow is triggered in the background:

  • Visual Analysis: The backend agent sends an optimized version of the image to the Vision model.
  • Metadata Generation: The AI generates a museum-grade Title, a Contextual Description (2-sentence historical or artistic blurb), and a set of Descriptive Tags.
  • Database Entry: This information is saved alongside your image, awaiting your final approval.

The 'Pending Review' State

To ensure accuracy and prevent AI "hallucinations" from appearing on your displays, all newly ingested art is placed in a Pending Review state. This acts as a human-in-the-loop safety mechanism.

  • Navigate to the Review Queue tab in the Admin Console.
  • Review the AI-generated titles and descriptions. You can edit any field to correct historical facts or artistic interpretations.
  • AI Guidance & Regeneration: If the AI completely misses the mark, you can provide context in the "AI Guidance" field (e.g., "This is actually a family photo from 1995") and click Regenerate. The agent will rebuild the metadata using your specific facts.
  • Click "Approve & Publish" to move the artwork into the live display rotation.
Developer Note: Swapping Models
While the system uses Gemini by default for its high performance-to-cost ratio, the backend is designed to be modular. Advanced users can swap to OpenAI (GPT-4o), Anthropic (Claude 3.5 Sonnet), or local models by modifying the agents.py logic and adding the respective API keys to your .env file.

URL Parameters Reference

Append these parameters to your display URL (e.g., http://SERVER:8000/?display=tv_01&rotate=true) to override localized behavior.

Parameter Description Example
rotate Forcefully rotates the entire DOM by -90 degrees. Useful for TVs mounted vertically where the OS remains in landscape mode. rotate=true
display Assigns a unique ID to the display. This ID allows the Mobile Remote to target this specific screen. Defaults to default. display=living_room
cycle_time The number of seconds between artwork transitions. cycle_time=60
mode Overrides the visual rendering mode. Options: ken-burns, static-crop, contain-matte. mode=contain-matte
shuffle Randomizes the playback order of the current playlist. Options: true, false. shuffle=true
placard_wait Seconds to wait after a new image loads before the info placard slides in. placard_wait=5
placard_show Seconds to keep the placard visible during its initial entrance. placard_show=20
placard_manual Seconds to display the placard when manually triggered by the remote or user interaction. placard_manual=15

The Mobile Remote Control

The remote console is accessible at /remote from any mobile browser on the same network.

  • WebSocket Routing: The remote uses WebSockets for instant control. When you change a playlist or mode on the remote, the targeted TV updates immediately without a page refresh.
  • Target Selection: Use the "Target Display" dropdown at the top of the remote to select which screen you want to control. Displays appear in this list as soon as they are opened in a browser.
  • Instant Actions: The remote provides dedicated buttons for Next/Previous artwork and a "Show Placard" button to quickly identify pieces.

Hardware & Kiosk Best Practices

For a seamless "Museum" experience, your display should behave like a native appliance rather than a browser window.

Recommended Browsers

  • Android / Fire OS: We highly recommend Fully Kiosk Browser. It allows you to lock the TV to a specific URL, hide all toolbars, and prevent the screen from sleeping.
  • Android TV: TV Bro is a great open-source alternative that handles full-screen web apps well on leanback devices.
Pro Tip: Most smart TVs have an "Auto Power Off" setting. Ensure this is disabled to prevent the artwork from stopping after a few hours of "inactivity."
